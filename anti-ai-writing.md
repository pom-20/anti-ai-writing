# AI臭さ除去スキル
 
AI生成文章を人間の文章に近づけるための編集ルール集。  
だ・である調のレポート文体を前提とするが、原則は他の文体にも転用可能。
 
---
 
## レベル1：基本除去チェックリスト
 
### 1. 文末のローテーション破壊
 
AIは同じ文末表現を繰り返す。
 
**NGパターン（検出されやすい）**
```
〜として注目されている。
〜として用いられている。
〜として期待されている。
```
 
**対処**  
同一段落内で文末が被らないよう意図的にバラける。  
「〜となっている」「〜の一つである」「〜が続いている」「〜を担う」など角度を変える。
 
---
 
### 2. 接続詞の機械的連打を止める
 
「また」「さらに」「加えて」の等間隔連打はAIの典型。
 
**対処**  
接続詞を省いて文と文を直結させるか、接続詞なしで改行する。  
どうしても必要な場合は1段落に1個まで。
 
---
 
### 3. 「特徴→用途」の定型構造を崩す
 
**AIの典型パターン**
```
〜という特性を持つ。そのため、〜に用いられる。さらに〜にも応用されている。
```
 
**対処**
- 用途から入って特性を後置する
- 逆接（「〜として知られているが、現在は〜」）で切り返す
- 段落ごとに切り口の順序を変える
---
 
### 4. 冗長な言い換えを削る
 
AIは同じ情報を微妙に言い換えて水増しする。
 
**対処**  
どちらか一方だけ残して切る。情報密度を上げるとAI臭さが下がる。
 
---
 
### 5. 数値・固有表現をそのまま刺す
 
「40%超」「5〜6倍」「ETL」などを説明で包まずに直接置く。  
AIは数値の後に必ず補足説明を付けるが、それが却って臭い。  
読み手を信頼して説明を省く。
 
---
 
### 6. カギカッコの多用を止める
 
AIは強調したい語を片っ端からカギカッコで囲む。
 
**NGパターン**
```
「ナノジェネレーター（環境発電デバイス）」の材料として研究されており、
外部電源を必要としない「次世代ウェアラブルデバイス」などへの応用が期待されている。
```
 
人間のレポートではカギカッコは初出の概念・定義の明示・引用にしか使わない。
 
**対処**  
カギカッコは1段落に1個まで。強調したいだけなら外す。初出の専門用語は括弧で英訳や読みを添える程度にとどめる。
 
---
 
### 7. 句読点の過剰な多用を止める
 
日本語AIは読点（、）を過剰に打つ。一文が長くなると読点で繋ぎ続ける。
 
**NGパターン**
```
光を照射することで強い酸化還元力を持つ光触媒として、建築物の防汚コーティングや、
空気清浄機のフィルターなどに、広く実用化されている。
```
 
**対処**  
1文に読点が3個以上あったら文を分割する。あるいは読点を1〜2個に間引いて締める。
 
---
 
## レベル2：人間模倣チェックリスト
 
### 8. 意図的な「微妙な言い淀み」を入れる
 
人間の文章には論理の継ぎ目がやや雑なところがある。AIは常に完璧に接続しようとするのでそれが臭い。
 
**実践**
```
〜として実用化が進んでいる。ただ、コスト面での課題は残る。
```
 
「ただ、」以降は内容的に不要でもよい。完璧に整合しすぎない文が人間らしさを作る。  
1〜2個/全体が目安。多用すると逆効果。
 
---
 
### 9. 書き手の視点を一瞬滲ませる
 
AIは常に俯瞰・客観で書く。人間は途中で主観が滲む。
 
**実践**  
「とみられる」「だろう」「といえる」など予測・見解の含みがある表現を1〜2箇所差し込む。  
書き手の人格が一瞬見えることで文章に体温が出る。
 
---
 
### 10. 段落の長さを意図的に不揃いにする
 
AIは段落をほぼ均等な長さに揃える。
 
**対処**  
2文で終わる段落と5文ある段落を混在させる。  
重要度に関係なくていい、リズムの問題。
 
---
 
### 11. 前提知識の説明を意図的に省く
 
AIは親切なので前提を全部説明する。人間は知っていることを省く。
 
**NGパターン**
```
直接遷移型のバンド構造を持つため、電子と正孔が直接再結合し、発光効率が高くなる。
```
 
**OKパターン**
```
直接遷移型のため発光効率が高く、〜
```
 
説明の省略が読み手への信頼感を演出する。
 
---
 
### 12. 同義語ローテーションをやめて構造ごと変える
 
「用いられる→活用される→採用される」のローテーションは検出器も人間も気づく。
 
**対処**  
表現を言い換えるのではなく、文の構造ごと変える。  
「〜が主流となっている」「〜での実績がある」「各社が〜を使い始めている」など角度を変える。
 
---
 
### 13. 能動と受動を混ぜる
 
AIは受動態に偏る（「用いられている」「期待されている」）。
 
**対処**  
意識的に能動文を差し込む。
 
```
研究者たちはこの材料を〜に使い始めている。
各社が採用を進めている。
```
 
主語を立てた能動文が人格感を出す。
 
---
 
### 14. 「逆張りの一文」で検出器を抜く
 
AI検出器はエントロピー（次の単語の予測しやすさ）で判定している。  
次の単語が予測しにくい文ほど人間判定される。
 
**実践**  
各段落に一文、やや唐突な切り口・比較・含みのある文を入れる。
 
```
もっとも、シリコンが今後も中心であり続けるかどうかは、製造コストの問題と切り離せない。
```
 
内容は正しくていい。ただし「次にこの文が来る」とAIが予測しにくい構造であることが重要。
 
---
 
## 適用手順
 
1. 原文を読み、レベル1チェックリストを全項目確認する
2. 修正箇所を洗い出してリライトする
3. レベル2チェックリストで追加の人間化を施す
4. 最後に通読して「読んで意外だった」と感じる文が各段落に1つあるか確認する
---
 
## まとめチェックリスト
 
**レベル1（基本）**
```
□ 同じ段落内で文末が被っていないか
□ また／さらに／加えて が3連続していないか
□ 特徴→用途の順番が全段落で同じになっていないか
□ 同じ内容を2文かけて言い換えていないか
□ 数値や固有名詞の直後に不要な説明を付けていないか
□ カギカッコが1段落に2個以上ないか（強調目的なら外す）
□ 1文に読点が3個以上ある文を分割または間引いているか
```
 
**レベル2（人間模倣）**
```
□ 論理的に不要だが自然な「ただ、」「もっとも、」が1〜2個あるか
□ 書き手の予測・見解が1箇所でも滲んでいるか（「とみられる」「だろう」）
□ 段落の長さがバラバラか（2文段落と5文段落が混在）
□ 前提知識の説明を意図的に省いている箇所があるか
□ 受動態が3連続していないか
□ 「読んで意外だった」と感じる角度の文が各段落に1つあるか
```
 
両方のリストをパスすれば、現行のAI検出器はほぼ抜けられる。
